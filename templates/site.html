One file, everything

A single HTML file contains the editor, your content, and all its history. Email it, put it on a USB stick, open it anywhere.

Version history

Commit, branch, and diff your content — just like Git, but embedded right inside the document itself.

Offline first

No internet connection required. No server. No account. Your data never leaves your device unless you choose to share the file.

Multiple formats

Markdown for notes, Mermaid for diagrams, ABC for sheet music, Fountain for screenplays — or all of them in one document.

Supported formats

Write in the format that fits your content

Each section of a document can use a different DSL — mix diagrams with prose, or sheet music with notes.

Markdown

built-in

GitHub-flavored Markdown with YAML front matter, page breaks, and live preview. The default format for notes, docs, and READMEs.

prose notes tables links

Mermaid

plugin

Diagrams and charts from plain text. Flowcharts, sequence diagrams, Gantt charts, ER diagrams, mind maps, and more.

flowchart sequence gantt ER

ABC Notation

plugin

Standard ABC notation for writing and rendering sheet music. Click notes to jump to the corresponding position in the editor.

sheet music playback export SVG

Fountain

plugin

Industry-standard Fountain markup for screenplays and scripts. Renders with correct formatting for scene headings, dialogue, and action.

screenplay script dialogue

Plugins are installed by drag-dropping a .plugin.js file onto the editor — no rebuild needed.

How it works

The quine principle

Every save rewrites the file so it includes both the updated content and the complete application. It always stays self-sufficient.

Step 01

Download once

Grab a single unifile.html. No installer, no account, no setup. Open it in any modern browser.

Step 02

Write your content

Use Markdown, Mermaid, ABC notation, or Fountain. Mix formats freely within the same document using #!dsl section markers.

Step 03

Commit and save

Commit your changes with a message. The file rewrites itself to include the new content and the commit in its embedded history.

Step 04

Share the file

Send the HTML file to anyone. They open it in a browser and see your document — with full edit and history access — instantly.
